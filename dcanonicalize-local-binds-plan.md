# Plan: Add `-dcanonicalize-local-binds` flag to GHC

## Context

Golden-file tests that check Core output via `-ddump-simpl` are prone to spurious failures because local binder names (`go1`, `go2`, `x1`, `ds`, `eta`, `ipv`) are generated by GHC's internal naming heuristics. These names can shift from seemingly unrelated compiler changes. `-dsuppress-uniques` doesn't fully help because System names always show their unique, and the base OccNames themselves can change.

The new `-dcanonicalize-local-binds` flag renames all local binders to canonical names based on order of first appearance, making Core dumps deterministic and robust against internal compiler changes.

## Approach

A Core-to-Core canonicalization pass applied to the bindings just before they are printed in `dumpPassResult`. External names (top-level binders, imports) remain unchanged. Each top-level binding is canonicalized independently.

### Canonicalization algorithm

For each top-level binding:
1. Walk the RHS in left-to-right, outside-in order, collecting local binders at their **binding sites**: lambda params (L→R), let/joinrec binders (definition order), case-alternative binders (alt order)
2. Skip binders with External names — these must stay unchanged
3. Skip type variables — less fragile, and hidden by `-dsuppress-type-signatures` anyway
4. Assign canonical names from a single namespace: `clbA`, `clbB`, ..., `clbZ`, `clbA1`, `clbB1`, ..., `clbZ1`, `clbA2`, ...
5. Dead binders → `_`
6. Build a `Subst` mapping old Ids → new Ids (same type/IdInfo, new name) and apply via `substBind`

### 1. Add `GeneralFlag` constructor

**File:** `compiler/GHC/Driver/Flags.hs` (~line 860)

Add `Opt_CanonicalizeLocalBinds` to the `GeneralFlag` enum (alphabetical order with other `Opt_*` entries).

### 2. Register the flag

```haskell

```

This auto-generates `-dcanonicalize-local-binds` and `-dno-canonicalize-local-binds`.

### 3. Add `sdocCanonicalizeLocalBinds` to `SDocContext`

Add field: `sdocCanonicalizeLocalBinds :: !Bool`

Also update `defaultSDocContext` to set it `False`.

### 4. Initialize from DynFlags

**File:** `compiler/GHC/Driver/DynFlags.hs` (~line 1586, in `initSDocContext`)

Add: `sdocCanonicalizeLocalBinds = gopt Opt_CanonicalizeLocalBinds dflags`

### 5. Create canonicalization module

**File:** `compiler/GHC/Core/Ppr/Canonicalize.hs` (new)

```haskell
module GHC.Core.Ppr.Canonicalize (canonicalizeBindsForPrinting) where

canonicalizeBindsForPrinting :: [CoreBind] -> [CoreBind]
```

Key implementation details:
- Use `GHC.Core.Subst` (`substBndr`, `substExpr`, `substBind`) for substitution
- Use `GHC.Types.Name` (`mkInternalName` or `setNameLoc`+`setNameOcc`) to create renamed Ids via `setIdName`
- Two-phase: (1) collect binders → (2) build subst → (3) apply subst via `substBind`
- Each top-level `CoreBind` is processed independently
- Only term-level Ids with `isInternalName` or `isSystemName` are renamed; External names and type vars are left alone

### 6. Hook into `dumpPassResult`

**File:** `compiler/GHC/Core/Lint.hs` (~line 373-392)

Add a `Bool` parameter for canonicalization to `dumpPassResult`:

```haskell
dumpPassResult :: Logger -> Bool -> Bool -> NamePprCtx -> Maybe DumpFlag
               -> String -> SDoc -> CoreProgram -> [CoreRule] -> IO ()
-- 2nd Bool = canonicalize_binds (new); 1st Bool = dump_core_sizes (existing)
```

In the `where` clause:

```haskell
final_binds
  | canonicalize_binds = canonicalizeBindsForPrinting binds
  | otherwise          = binds
```

Then use `final_binds` instead of `binds` in `pprCoreBindings` / `pprCoreBindingsWithSize`.

Update both call sites to pass the flag:
- `endPassIO` in the same file (~line 352): extract from `SDocContext` via `sdocCanonicalizeLocalBinds (log_default_dump_context (logFlags logger))`
- `dump_end_iteration` in `compiler/GHC/Core/Opt/Simplify.hs` (~line 288): same approach

### 7. Update the elem tests

**File:** `libraries/base/tests/perf/all.T` (~line 15)

Add `-dcanonicalize-local-binds` to the compilation flags string.

Then update the 4 golden `.stderr` files to use canonical names (e.g., `x`→`a`, `eta`→`b`, `go1`→`c`, `ds`→`d`, `y`→`e`, `ys`→`f`, etc.).
